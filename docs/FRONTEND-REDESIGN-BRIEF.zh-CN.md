# 个人站前端重设计 Brief

这份文档是给新的前端设计 / 实现模型看的。

目标不是“简单美化一下”。

目标是：

- 在不破坏网站定位的前提下，重做第一版视觉和前端呈现
- 让它更像一个真实工程师的个人站
- 同时兼顾 `博客 + 简历 + 项目展示`

## 项目背景

这是一个个人网站，不是产品官网。

站点主人设是：

`一个 10 年 Java 后端，经历过大厂裁员，现在在转 AI 应用和独立开发。`

网站要承载四件事：

- 公开写作
- 个人品牌
- 面试友好的公开简历
- 项目展示

## 当前问题

当前版本的问题不是“不能用”。

当前版本的问题是：

- 视觉气质太平
- 首页冲击力不够
- 个人身份没有被真正立住
- 太像普通模板站的初稿
- 还没有形成自己的审美语言

## 这次要的不是

不要做成这些方向：

- 通用 SaaS landing page
- 白底紫按钮的 AI 模板站
- 花哨但空的作品集模板
- 过度科技感的霓虹大屏
- 一堆动画，但信息层次没处理好的站

## 这次真正想要的方向

更接近这几个关键词：

- 冷静
- 克制
- 有工程感
- 有身份感
- 阅读友好
- 不端着
- 不油腻

可以有设计感，但不能像假创业者主页。

## 设计目标

访问者打开首页后，前几秒内应该能感受到：

- 这是一个真实程序员的站
- 这个人有明确经历，不是空人设
- 他在写东西，也在做东西
- 这个站既能读文章，也能看简历和项目

## 非协商信息架构

下面这些页面必须保留：

- `Home`
- `Writing`
- `Projects`
- `About`
- `Resume`
- `Now`

这次可以重做：

- 首页布局
- 导航样式
- 卡片系统
- 字体选择
- 色彩系统
- spacing
- 响应式细节
- 页面节奏

这次不要擅自改掉：

- 页面路由
- 网站定位
- 内容集合结构
- 文章 / 项目 / 简历 三条主线

## 首页重点

首页不是文章列表首页。

首页的第一任务是立住身份。

建议优先强化这些模块：

1. 首屏身份区
2. 精选文章
3. 代表项目
4. 当前状态 / 当前关注
5. 导向 Resume 和 About 的入口

## 内容气质要求

这站不是“我做过很多厉害东西”的炫耀站。

它更像：

`一个处在转向中的工程师，把自己的判断、写作、项目和路径公开放出来。`

所以视觉上要帮助这个感觉成立：

- 有专业度
- 有真实感
- 有长期写作的人味

## 视觉建议

### 字体

不要默认系统无特色排版。

优先考虑：

- 英文标题和数字有一点 personality
- 中文正文仍然保证长文可读

### 配色

不要紫色路线。

可以考虑：

- 偏暖灰底
- 深色文字
- 一个有克制感的强调色
- 卡片和背景有层次，但不要太花

### 布局

不要每个模块都长得一样。

首页至少应该有：

- 一个有记忆点的 hero
- 一个区别于普通博客列表的 featured 区
- 一个更像“公开工作台”的项目区

### 动效

只要少量有效动效。

可以有：

- 首屏进入
- 卡片 hover
- 页面区块渐进出现

不要堆一堆炫技 motion。

## 技术约束

当前技术栈：

- `Astro`
- Markdown content collections

请尽量：

- 复用现有内容结构
- 不要引入很重的前端依赖
- 不要为了视觉重构成复杂客户端应用

可以改：

- `src/layouts/`
- `src/pages/`
- `src/styles/`

必要时也可以新增：

- `src/components/`

## 产出要求

希望你给出的不是“建议清单”，而是能直接落地的前端实现。

最低交付：

- 重做首页
- 统一全站视觉系统
- 优化导航和卡片
- 让 `Writing / Projects / About / Resume / Now` 页面风格统一

最好顺手处理：

- 更好的 section heading
- 更像个人站的 footer
- 更像真实文章站的阅读页样式

## 验收标准

最终效果至少要满足：

- 一眼不像模板站
- 一眼不像资讯站
- 一眼能看出是技术人的站
- 阅读页舒服
- 首页更有记忆点
- Resume 页面更像“会让面试官愿意继续看”的页面
